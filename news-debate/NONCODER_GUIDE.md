# News Debate App: Plain-English Guide

## What this app does

This app lets you type in a debate topic, such as:

- `Should AI tools be used in classrooms?`
- `Should remote work be mandatory?`

It then creates a short debate with:

- arguments in favor
- arguments against
- final rebuttals from both sides

Think of it as a simple debate simulator.

## Two ways to use it

### 1. Terminal version

This is the text-only version.

You run:

```bash
python main.py
```

Then the app asks for a topic and prints the debate in the terminal window.

### 2. Browser version

This is the visual version.

You run:

```bash
python web_app.py
```

Then you open this address in your browser:

`http://127.0.0.1:8000`

This version gives you a page with a text box, a button, and styled debate cards.

## What the page is showing

The browser version has a few simple parts:

- A title at the top that explains the app.
- A short description telling you it runs in local mock mode.
- A text box where you type the debate topic.
- A `Generate Debate` button that sends your topic to the app.
- A list of result cards underneath.

Each result card shows:

- the round number
- whether it is `For` or `Against`
- the actual argument text

The green cards are the supporting side.

The red cards are the opposing side.

The page background, card colors, spacing, and typography were added to make the app easier to read and feel more like a small product instead of raw text.

## What “local mock mode” means

This app can work in two different ways:

### OpenAI mode

If you have a real OpenAI API key, the app can send requests to OpenAI and get model-generated responses.

### Local mock mode

If you do not have an API key, the app still works offline.

In local mock mode:

- nothing is sent to OpenAI
- no internet connection is required for the debate engine
- the app generates reasoned pro and con arguments using local Python code

Right now, your app is using local mock mode.

## What `127.0.0.1:8000` means

This address looks technical, but it is simple once broken down:

- `127.0.0.1` means “this same computer”
- it is also called `localhost`
- it does **not** mean a public website on the internet
- only someone on your machine can open it

The `:8000` part is the port number.

A port is just a numbered doorway that lets one program talk to another on the same computer.

So:

- `127.0.0.1` = your own machine
- `8000` = the specific doorway used by this app

## What server was running

When you started the browser version with:

```bash
python web_app.py
```

Python launched a small built-in web server called `HTTPServer`.

That server does three important jobs:

1. It keeps listening for browser requests.
2. It sends the web page to the browser.
3. It receives the topic you typed and generates the debate results.

This is the process that “keeps the app open.”

More precisely:

- the browser page stays available because the Python server keeps running
- each time you submit a topic, the server processes it and returns a fresh page

## Important clarification about “AI staying open”

There is not a separate always-running AI model inside this app.

What stays running is the local Python web server.

When you submit a topic:

- the server receives the topic
- the debate code runs
- the results are turned into HTML
- the browser shows the updated page

In your current setup, the reasoning is being generated by local Python logic, not by a live cloud AI service.

## How the app works behind the scenes

Here is the app in very simple steps:

1. You open the page in the browser.
2. The browser talks to the local Python server.
3. You enter a topic and press the button.
4. The server sends that topic into the debate engine.
5. The debate engine creates pro and con arguments.
6. The server builds a new HTML page with those results.
7. The browser displays the formatted debate cards.

## Main files in the project

These are the most important files:

- `main.py`
  - runs the terminal version
- `web_app.py`
  - runs the browser version and local web server
- `debate.py`
  - creates the pro/con arguments
- `README.md`
  - quick setup and run instructions

## Why the app still works without an API key

Originally, the app expected an OpenAI API key and would stop with an error if the key was missing.

That was changed so the app can now:

- detect whether a key exists
- use OpenAI if a key is available
- otherwise fall back to the local reasoning mode

This makes the app easier to demo, test, and understand without paid API access.

## What is not happening

A few things are easy to assume but are not true here:

- the browser is not connected to a public website
- the app is not exposing itself to the internet
- the app is not running a separate database
- the app is not keeping a conversation memory between topics
- the app is not continuously streaming model output

Each topic is handled as a fresh request.

## If you want to explain it in one sentence

This is a local debate generator that runs on your own computer, shows a simple browser interface, and uses a Python server to turn a topic into readable pro and con arguments.
